# Persistent run history with transcript replay

## Metadata
- **Category**: functionality
- **Effort**: Unknown (6/3)

## Description
CLI executions today live only in an in-memory Map (activeExecutions in cli-service.ts, GC- d after 1h) and raw logs are dumped to .claude/logs/ with no UI. Persist each run to the existing better-sqlite3 DB (id, task type, module, prompt, cost, duration, success, file changes, full event stream) and add a Runs panel to browse, search, and replay past transcripts. Link every run to the checklist item / feature / catalog entity it advanced via the staticFields already attached to callbacks.

## Reasoning
Every serious agent and CI tool (GitHub Actions, CircleCI, Devin sessions, Cursor) keeps durable run history because users need to answer what did the agent do last night and re-open a finished run. Right now a 40-minute live run vanishes on restart, so debugging regressions or auditing what changed is impossible. One step further than CI: because PoF knows which checklist/feature each run targeted, the history doubles as a build ledger of how the game was assembled.
