# Self-balancing loot economy from live playtest telemetry

## Metadata
- **Category**: functionality
- **Effort**: Unknown (9/3)

## Description
Close the loop between PIE and the simulator: stream real playtest telemetry (drops, kills, currency earned, time-to-upgrade) back from UE5, compare it against designer balance targets (e.g. "a Legendary every 45 min"), and have the system detect drift and auto-propose weight adjustments that re-inject into the running game via the existing ue5-inject-item bridge. Over many sessions the loot tables tune themselves toward the intended pacing. First sprint step: define a balance-target config plus a telemetry ingestion endpoint, and chart observed-vs-target drop cadence in the LootTableVisualizer economy tab.

## Reasoning
No competing tool turns live player behavior into automatic economy tuning; it would make PoF the definitive ARPG balance platform and a compounding strategic asset as telemetry accumulates. It eliminates the endless manual playtest-then-retune cycle that defines loot design today. The drop-simulator and live-inject route already provide both halves of the loop.
