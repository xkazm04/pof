# Value vs. Effort Prioritization Quadrant View

## Description
Add a 2x2 scatter view plotting each plan item by impact.score (Y axis) against effort.minutes (X axis), dividing features into Quick Wins, Big Bets, Fill-ins, and Time Sinks quadrants. Reuse the existing useImplementationPlan data and the matrix-map rendering infrastructure. Ready items render solid while dependency-blocked items appear ghosted, and clicking a point fires the existing Implement This handler.

## Reasoning
Value-vs-effort matrices are the standard prioritization visual in ProductBoard, Aha!, and Jira because they make trade-offs instantly legible. The impact and effort numbers already exist but are buried in list badges; surfacing them spatially turns prioritization into a single glance. Effort is low since it reuses existing plan generation.
