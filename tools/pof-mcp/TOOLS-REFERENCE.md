### `pof_submit_artifact`

Submit the work you produced for a step (data object + UE asset paths). The SERVER derives the acceptance verdict from the step's own checker — you never self-grade. On fail, read the reason and retry; L3/L4 "deferred" is upgraded later by pof_drain_gates.

**Arguments:**
- `catalogId` **(required)**: string
- `entityId` **(required)**: string
- `step` **(required)**: string
- `data` **(required)**: object — The produced artifact data.
- `ueAssets`: array — UE content paths created (e.g. /Game/Items/Sword/T_Sword_Icon).

> No static example — write tool — recorded by the quality walker with a test-namespaced entity

### `pof_drain_gates`

Run the deferred L3/L4 Test Gates for an entity against the LIVE UE editor (via the PoF bridge), turning "deferred" gates into pass/fail. The editor is non-reentrant — one drain at a time (concurrent → 409). Set allowSpawn to use a headless editor.

**Arguments:**
- `catalogId` **(required)**: string
- `entityId` **(required)**: string
- `tier`: string
- `allowSpawn`: boolean

> No static example — needs a live UE editor — recorded by the live growth suite

## Harness loop

### `pof_harness_start`

Start the autonomous harness loop (plan → execute → verify → checkpoint) for a UE project. Returns immediately; poll pof_harness_status. Use this to push a whole game forward, not a single step.

**Arguments:**
- `projectPath` **(required)**: string — Absolute path to the UE project root.
- `projectName` **(required)**: string
- `ueVersion` **(required)**: string — e.g. "5.7".
- `maxIterations`: number
- `targetPassRate`: number — 0–1; stop when this feature pass-rate is reached.
- `budgetUsd`: number — Spend cap; the loop auto-pauses on overflow.
- `checkpoint`: boolean — Git-checkpoint each completed area.

> No static example — launches the autonomous loop — recorded by a bounded control-plane case
