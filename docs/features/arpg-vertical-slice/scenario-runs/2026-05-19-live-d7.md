### Root cause hypothesis for the flake

`useModuleCLI.sendPrompt` at `src/hooks/useModuleCLI.ts:122-128` waits `UI_TIMEOUTS.mountDelay` (100ms — verified `src/lib/constants.ts:69`) after `setActiveTab(tabId)` before dispatching the event. The CompactTerminal needs to mount and register its bubble-phase listener within that window. If the mount is slow (e.g., first session for a new module under load), CompactTerminal hasn't registered yet when the event fires.

Our addInitScript listener catches the event (registered at page load on `window` capture phase, always present), records it to the buffer, and passes it through (live mode → no `stopImmediatePropagation`). But if CompactTerminal isn't listening yet, the event has nothing to propagate to. The Claude CLI session never starts. The store's `isRunning` stays false. The `pof-cli-panel-running-indicator` testId never renders.

In D7's first run, the timing happened to align — CompactTerminal was mounted in time. In D7.5, it wasn't.
