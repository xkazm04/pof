## Part C — arena re-texture via the 3D-texture endpoint ⚠️ (endpoint unusable)

- **`export_arena_obj.py`** → `Content/ArenaBuild/Arena.obj` via
  `StaticMeshExporterOBJ`. UV-mapped: 1668 verts / 1668 UVs / 556 faces; face
  lines are `f v/vt` triplets. **Works.**
- **3D-texture endpoint — NOT usable.** Live API discovery (2026-05-23):
  - `POST /models-3d/upload` takes **`modelExtension: "obj"`** (not `modelType`),
    and returns an **S3 presigned multipart POST** (`modelUrl` + `modelFields`
    JSON), **not** a PUT. The OBJ upload **works** (S3 returns 204; the model is
    retrievable at `GET /models-3d/{id}` with a `meshUrl`).
  - **`POST /generations-texture` returns 404** ("Endpoint not found"). Probed
    ~12 candidate create paths; `/models-3d/texture` etc. only resolve as
    `/models-3d/{id}` (GET, DELETE). **There is no public endpoint to GENERATE a
    PBR texture set on an uploaded model.** The capability the
    `leonardo-api-capabilities` memory recorded as "legacy-flagged but live" has
    been removed from the public REST API — only 3D model upload + retrieval +
    deletion remain.
  - **`generateTextureOn3DModel` corrected** to the real upload flow (it uploads
    successfully, then throws a clear error at the 404 create step, and deletes
    the uploaded model on cleanup). All test models created during discovery were
    `DELETE`d (account left clean).
- **Outcome (honest fallback per spec):** no 3D PBR set was produced, so
  `retexture_arena_3d.py` was not written/run and the Gemini A/B was N/A. The
  arena **keeps PS-3's textures**, now served through `M_ARPG_Surface_Master` via
  the `MI_Arena_*` instances (Part B). Parts A + B ship full value.
