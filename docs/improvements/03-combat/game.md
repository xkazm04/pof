# 03 · Combat — Game Improvements

## Goals

Move combat from "the ability activates and damage applies via a test-only
event send" to "you press LMB, you see a swing, the enemy reacts, you feel
the hit." Plus: granting the rest of the abilities the project has
generated but never wired.

## Improvements

### 1. `bUseAnimationDrivenDamage` toggle on `UGA_MeleeAttack`

Add a `UPROPERTY(EditDefaultsOnly) bool bUseAnimationDrivenDamage = false`
on `UGA_MeleeAttack`. When `false` — applied at a deterministic offset
inside `ActivateAbility` (or after the existing `WaitDelay` fallback) — the
ability is gray-box-correct without needing a montage notify. When `true`,
the existing `WaitGameplayEvent(Event.MeleeHit)` listener handles damage
normally. PoF's combat module ships with `false`; flipping to `true` is the
"add real animation" follow-up. This retires the PS-1 test-sends-the-event
hack.

### 2. Real attack / dodge / hit-react / death animations

Cross-references [[../02-character/game.md]] §1 — once a real `AM_MeleeCombo`
exists on the mannequin skeleton, wire `AnimNotifyState_HitDetection` to
fire `Event.MeleeHit` and set `bUseAnimationDrivenDamage = true` on
`BP_GA_MeleeAttack`. Same shape for:
- `AM_Dodge_*` driven by `GA_Dodge` (the project has 4 directional dodge
  montages — also empty shells).
- `AM_HitReact` driven by `GA_HitReact` — plays when the player takes a
  hit.
- `AM_Death` driven by `GA_Death` — plays before `SetLifeSpan` fires.

Once these are real, the slice has a visibly responsive combat loop.

### 3. Grant + wire the other generated abilities on the player

The project has `GA_DashStrike`, `GA_Fireball`, `GA_ArcaneBeam`,
`GA_GroundSlam`, `GA_WarCry`, `GA_UsePotion` — all generated by SP-B,
none granted to the player. Add them to `BP_VSPlayer`'s `DefaultAbilities`
and bind their input actions in `ARPGPlayerController` (`IA_AbilitySlot1..4`
already exist as the controller's UPROPERTYs). The slice's "one ability"
becomes a more recognisable RPG hotbar.

For each, the same `bUseAnimationDrivenDamage` toggle decides whether the
montage is required.

### 4. A real damage formula in `UARPGDamageExecution`

The existing `UARPGDamageExecution` likely reads `Base`. Extend it to a
small but real formula:
- `final_damage = max(0, base * (1 + attack_power_bonus) - target_armor) *
  (crit ? crit_multiplier : 1)`
- `crit = random < target.crit_chance_resist_adjusted`
- elemental: if the GE has a `Damage.Fire/Ice/Lightning` tag, apply the
  matching resistance reduction.

The values come from the `DT_AttributeDefaults` DataTable
([[pof-app.md]] §4). Until that DataTable exists everything except `Base`
is zero and the formula degenerates correctly to "raw base damage."

### 5. The player-Health double-bookkeeping resolution

`AARPGPlayerCharacter` has both plain-float `Health`/`MaxHealth` and the
GAS `Health`. Pick a side:
- **Recommended:** keep GAS as the source of truth (the HUD reads it,
  damage applies through it). Make `AARPGPlayerCharacter::GetHealth()` /
  `GetMaxHealth()` / `GetHealthPercent()` read GAS via the ASC; deprecate
  the float setters, or sync them from `PostGameplayEffectExecute` if
  any consumer needs them.

Document the decision; remove the parallel state to eliminate a future
"why is the HUD saying 80 but Blueprint thinks 100" hour.

### 6. Combat feel — minimal hit pause + camera shake

Two small wins from the combat-feel polish list, both pure C++:
- **Hit pause:** in `GA_MeleeAttack::OnMeleeHit`, before applying damage,
  `UGameplayStatics::SetGlobalTimeDilation(this, 0.1f)` for ~80 ms then
  back to 1.0 — a half-frame freeze that reads as impact.
- **Camera shake:** spawn an existing `UMatineeCameraShake` (UE has
  built-ins) on the player camera. The project already has
  `AnimNotify_CameraShake`; firing it directly from C++ when no notify
  exists works.

The slice's combat goes from "the bar dropped" to "the hit *landed*."

### 7. Death visual — ragdoll, fade-out

`ARPGCharacterBase::EnableRagdoll()` already exists (the inventory found
it). Call it from `OnDeathFromAbility` before `SetLifeSpan`, and add a
short fade-out (a translucent material instance MID) over the lifespan
so the corpse doesn't just vanish.

## Verification this work succeeded

- The PS-1 functional test still passes; a new "animation-driven path"
  variant of #4 — toggle `bUseAnimationDrivenDamage = true` and verify
  damage applies via the notify-fired event — also passes.
- The player can use at least three abilities (melee, dodge, one
  ranged/AoE); each visibly reacts and the HUD bars update.
- Hit pause + camera shake fire on enemy hits.
- Enemy death plays the death montage + ragdolls + fades out.
