The map's value compounds — keep entries thin.

Headless-per-run; ad-hoc; in-app UI. Default scenario input now `action:/Game/Input/Actions/IA_Move value:[0,1]` (documented-preferred over Boolean-key).
- **RESOLVED 2026-06-19 (was a low-displacement gap):** headless Move-injection drove ~10u with `speed 0`/`anim_speed 0` — diagnosed NOT as injection fidelity but **combat interference**: VerticalSlice enemies damage the player → `ARPGCharacterBase::CanMove()` false (stagger/hit-react) → `HandleMove` drops the injected movement. Fix = a **`disable_ai` scenario flag** (`UScenarioController` destroys all AI-possessed pawns at start, class-agnostic via `AAIController`; UE commit `c8111f9` on `feature/arpg-movement-feel`, editor rebuilt), plumbed `ScenarioSpec.disableAI → buildScenarioInbox → "disable_ai"` + UI "Remove enemies" toggle (default on). **Proven live:** forward walk → **1241u @ 600 speed, health flat 100** (was ~10u / speed 0 / health dropping). Build note: UBA's low-memory heuristic can loop-kill compiles when system RAM is cache-heavy → rebuild with `-NoUBA`.
