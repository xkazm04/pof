Create `src/app/prototype/page.tsx`. This is a Next.js App Router page. Since it uses hooks (useState, useRef, useFeatureMatrix), it needs `'use client'` directive.

**Page structure:**
1. **Header:** Minimal -- title "Dzin Prototype" in large text, nothing else in header row
2. **Control bar:** Single row below header with mode toggle and density/resize controls
3. **Layout container:** DzinLayout wrapping the Core panel

**Data wiring (per locked decision: real hooks from start):**
```typescript
const { features, isLoading } = useFeatureMatrix('arpg-combat');
const defs = MODULE_FEATURE_DEFINITIONS['arpg-combat'] ?? [];
const featureMap = useMemo(() => {
  const map = new Map<string, FeatureRow>();
  for (const f of features) map.set(f.featureName, f);
  return map;
}, [features]);
```
Import MODULE_FEATURE_DEFINITIONS from `@/lib/feature-definitions`.
Show a loading state (simple "Loading..." text or spinner) while `isLoading` is true.

**Control bar (per locked decision):**
- Mode toggle: two buttons "Override" and "Resize", styled as segmented control (active button highlighted)
- State: `mode: 'override' | 'resize'`, `selectedDensity: PanelDensity` (default 'full'), `selectedPreset: keyof RESIZE_PRESETS` (default 'Large')
- Override mode shows: 3 density buttons (micro / compact / full) -- clicking sets `selectedDensity`
- Resize mode shows: 3 preset buttons (Small / Medium / Large) -- clicking sets container width
- Styling: use PoF dark theme classes (bg-surface, border-border, text-text, etc.). Active button gets accent highlight (bg-blue-500/20 border-blue-500/50 or similar)

**Resize presets (Claude's discretion per CONTEXT.md):**
```typescript
const RESIZE_PRESETS = {
  Small: 160,   // Below compact minWidth 200 -> triggers micro
  Medium: 320,  // Above 200, below full minWidth 400 -> triggers compact
  Large: 800,   // Above 400 -> triggers full
} as const;
```

**DzinLayout integration:**
- Create a container div with ref (`containerRef = useRef<HTMLDivElement>(null)`)
- In resize mode: set container div `style={{ width: RESIZE_PRESETS[selectedPreset] }}` with a transition for smooth resize
- In override mode: container div has no width constraint (fills available space)
- PanelDirective: `{ type: 'arpg-combat-core', density: mode === 'override' ? selectedDensity : undefined }`
  - When density is undefined, Dzin auto-assigns based on container dimensions
- Pass `options={{ containerRef, preferredTemplate: 'single' }}` to DzinLayout so ResizeObserver measures the container, not the viewport (Pitfall 1 from research)

**renderPanel callback:**
```typescript
renderPanel={(assignment) => {
  const def = pofRegistry.get(assignment.panelType);
  if (!def) return null;
  return <CorePanel featureMap={featureMap} defs={defs} />;
}}
```
Note: For Phase 2 with a single panel type, we can directly render CorePanel. In Phase 3 this will become dynamic.

**Layout styling:**
- Page: min-h-screen bg-surface-deep p-6
- Container: centered, max-w-4xl for override mode, explicit width for resize mode
- Add a subtle label showing current active density below the layout (e.g., "Current density: compact") -- useful for testing even though user said no debug overlay. Keep it minimal: one line of text-xs text-text-muted. This is optional -- skip if it feels like debug info.

**No navigation link (per locked decision):** This page is accessed via direct URL `/prototype` only.

**Pitfalls to handle:**
- Pitfall 2: In resize mode, ensure `directive.density` is `undefined` (not falsy), so auto-density works
- Pitfall 3: Show loading state while useFeatureMatrix seeds data on first load
- Pitfall 5: Add 'use client' directive since page uses hooks
