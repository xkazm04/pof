  <name>Task 1: Rewrite prototype page with multi-panel layout, template picker, preset switcher, and FLIP animations</name>
  <files>src/app/prototype/page.tsx</files>
  <action>
Rewrite the prototype page completely. The page currently shows a single CorePanel with override/resize controls. Replace with a full multi-panel workspace.

**State management:**
- `templateId` state (LayoutTemplateId) -- initialized from first preset's templateId
- `activePresetId` state (string | null) -- tracks which preset is active (null if user manually switched template)
- `directives` state (PanelDirective[]) -- initialized from first preset's directives
- Keep existing `mode` (override/resize), `selectedDensity`, `selectedPreset` (resize preset), `containerRef`
- Keep existing data wiring (useFeatureMatrix, MODULE_FEATURE_DEFINITIONS, featureMap)

**Control bar layout** (extend existing controls):
After the existing Override/Resize toggle and density/size buttons, add a separator, then:
1. **Template picker:** 4 buttons showing minimap SVG thumbnails for split-2, grid-4, primary-sidebar, studio. Each is ~24x24px SVG showing colored rectangles representing slot proportions. Active template has accent border (`border-blue-500/50`) + subtle glow (`shadow-[0_0_6px_rgba(59,130,246,0.3)]`). Hover shows `border-border/80`. Clicking sets templateId and clears activePresetId to null (manual template selection breaks preset association).
2. **Preset dropdown:** Button showing "Preset: {label}" with a dropdown on click. Lists all 3 COMPOSITION_PRESETS. Active preset has a checkmark icon (Check from lucide-react). Selecting a preset sets templateId, directives, AND activePresetId all at once. Use a simple `useState<boolean>` for dropdown open state with an outer click handler to close.

**Template thumbnails SVGs** (Claude's discretion on exact design):
- split-2: Two rectangles side by side (~65%/35% proportions)
- grid-4: Four equal rectangles in 2x2 grid
- primary-sidebar: Large left rectangle + narrow right rectangle
- studio: Large top-left + 4 smaller slots arranged around it

**Multi-panel renderPanel dispatch:**
Create a `renderPanel` callback that switches on `assignment.panelType` and returns the correct panel component. Import all 10 panel components. Pass `featureMap` and `defs` as props to each.

**DzinLayout integration:**
Pass `directives`, `pofRegistry`, `renderPanel`, and `options={{ containerRef, preferredTemplate: templateId }}` to DzinLayout. When in override mode, map directives to include the selectedDensity.

**LayoutGroup + FLIP animations (LAYT-04):**
The DzinLayout's internal rendering needs to be wrapped. Since DzinLayout handles grid rendering internally, the animation wrapping needs to happen in the `renderPanel` callback. Each panel returned from renderPanel should be wrapped in a `motion.div` with:
- `layout` prop (enables FLIP)
- `layoutId={assignment.panelType}` (key by panel type, NOT slot index per anti-pattern warning)
- `initial={{ opacity: 0, scale: 0.95 }}`
- `animate={{ opacity: 1, scale: 1 }}`
- `exit={{ opacity: 0, scale: 0.95 }}`
- `transition={{ duration: DZIN_TIMING.LAYOUT, ease: LAYOUT_EASE }}`

Wrap the entire DzinLayout in `<LayoutGroup>`. Wrap DzinLayout's grid area with `<AnimatePresence mode="popLayout">` (NOT mode="wait" -- allows simultaneous layout animations). NOTE: If DzinLayout does not expose the grid container for wrapping, wrap the DzinLayout itself in LayoutGroup and apply motion.div inside renderPanel.

**DzinSelectionProvider:**
Wrap the entire layout section (below control bar) with `<DzinSelectionProvider>` so all panels share selection context.

**Clear selection on preset/directive change:**
Add a `useEffect` watching `directives` that calls `setSelection(null)` to avoid stale selection referencing panels no longer visible (Pitfall 5 from research).

**Styling:**
- Keep existing dark theme styling
- Give layout container explicit `min-h-[600px]` to prevent height collapse during template switch (Pitfall 2)
- Use `flex-1` in flex column layout for the main content area

Use `@/` imports. No raw console. No hardcoded hex colors (use Tailwind classes or CSS vars).
